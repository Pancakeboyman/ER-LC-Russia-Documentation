# 🚧 Коды Ошибок

<h3 align="center">Формат Ошибки</h3>

Большинство ошибок возвращаются в формате JSON с полем `detail`, описывающим проблему.

```json
{
    "detail": "текст_ошибки"
}
```

<p align="center"><strong>Что делать при неизвестной ошибке (код 0)?</strong></p>

Пожалуйста, **откройте тикет** и предоставьте следующую информацию:

* Какой эндпоинт вы вызывали.
* Какие параметры передавали (кроме секретных данных, таких как токены).
* Время возникновения ошибки.
* Полученный ответ от API.

***

<h3 align="center">Ошибки Аутентификации и Доступа</h3>

| Название Эндпоинта | Код | Статус       | Описание        | Причина и решение                                                                                                         |
| ------------------ | --- | ------------ | --------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `erlc/server/*`    | 401 | Unauthorized | `missing_token` | В запросе отсутствует API ключ. Передайте его в заголовке `Authorization: Bearer <ключ>`.                                 |
| `erlc/server/*`    | 401 | Unauthorized | `invalid_token` | Переданный API ключ недействителен, не найден или деактивирован. Убедитесь, что вы используете правильный, активный ключ. |
| `erlc/server/*`    | 403 | Forbidden    | `forbidden`     | Ваш API ключ не имеет разрешения на доступ к этому эндпоинту. Обратитесь к администратору для получения доступа.          |

<h3 align="center">Ошибки На Стороне Сервера</h3>

| Код | Статус                | Описание       | Причина и решение                                                                                                                          |
| --- | --------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 500 | Internal Server Error | `server_error` | Произошла ошибка при обращении к внешнему API ERLC. Внешний сервис может быть недоступен. API может вернуть данные из кеша, если они есть. |
| 502 | Bad Gateway           | `server_error` | Произошла ошибка при обращении к серверу. Сервер отключен или недоступен.                                                                  |
